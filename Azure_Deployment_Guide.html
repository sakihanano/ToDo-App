Azure 接続・デプロイ入門

1. 前提準備

Azure環境に接続する前に、以下をインストール・確認してください。

ツール用途インストール

社内ネットワーク上の注意

プロキシ環境では Azure CLI のSSL検証が失敗する場合があります。curl.exe を使う場合は --ssl-no-revoke を付与してください。

2. Azureへの接続

Azure CLIでログインし、正しいサブスクリプションに接続します。

Step 1: ログイン

az login

ブラウザが開くので、社用アカウント（@baycurrent.co.jp）でサインインしてください。

Step 2: サブスクリプション確認

az account show --query "{subscription:name, id:id, user:user.name}" -o table

以下が表示されれば正常です。

項目期待値
SubscriptionMHC_mhc-2026-6621
Userあなたの @baycurrent.co.jp アドレス

Step 3: サブスクリプションが違う場合

az account set --subscription "MHC_mhc-2026-6621"
権限がない場合

サブスクリプションが表示されない場合は、Azure管理者にContributor権限の付与を依頼してください。

4. デプロイ手順（Container Apps）

この環境ではAzure Container Appsへのデプロイが基本です。ACRでビルドし、Container Appを更新します。

1

ACRでイメージビルド

ローカルのDockerfileをAzure Container Registry（ACR）でクラウドビルドします。

az acr build --registry <ACR名> --image <イメージ:タグ> --file Dockerfile .
2

Container App更新

ビルドしたイメージでContainer Appを更新します。

az containerapp update --name <App名> --resource-group <RG名> --image <ACR>.azurecr.io/<イメージ:タグ>
3

動作確認

更新後にエンドポイントの応答を確認します。

Invoke-WebRequest -Uri "https://<FQDN>" -UseBasicParsing

実例：遊休資産管理のデプロイ

既存スクリプトを使った手順です。

Set-Location -LiteralPath "Z:\OneDrive - 株式会社ベイカレント\home\MHC\デジ戦企\Azure\scripts"

.\deploy-yushisan-containerapp.ps1 `
  -ResourceGroup "rg-mhc-nonprod-jpe" `
  -AcrName "cad8c0d7e5b6acr" `
  -ContainerAppName "ca-mhc-yushisan-np-0601" `
  -EnvironmentName "cae-mhc-nonprod-jpe" `
  -BasicAuthUsername "preview" `
  -BasicAuthPassword "<変更後のパスワード>"
Windows cp932ターミナルの注意

az acr build のログストリーミングでUnicodeエラーが出る場合は --no-logs を追加してください。

az acr build --registry <ACR名> --image <イメージ:タグ> --no-logs .

6. 守るルール

Azure環境を安全に使うために、全メンバーが守るルールです。

セキュリティ

  • APIキー、パスワード、Client Secret はチャット、Markdown、Git管理ファイルに書かない。
  • シークレットは Key Vault、Dify UI、Azure Portal に直接入力する。
  • キー取得コマンドは値を表示するため、画面共有・ログ保存に注意する。
  • az group delete、NSG全公開、DB firewall全開放は事前確認する。

運用

  • リソース削除、スケール変更、公開範囲変更は関係者へ事前連絡する。
  • IaC/スクリプトを正本とし、ポータル手作業は緊急時のみ。
  • デプロイ後は必ずエンドポイントの動作確認を行う。
  • Docker Compose変更後は Dify 画面と API を確認する。
  • node_modules やビルド成果物はローカルに溜めない。
最重要ルール

API Key、Azure OpenAI Key、パスワード、Token は、チャットや資料に絶対に記載しないでください。
